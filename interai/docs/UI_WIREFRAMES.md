# InterviewCopilot UI Wireframes

## Overview

This document provides textual descriptions of the UI components and layouts for InterviewCopilot. The design prioritizes minimal distraction, quick glance-ability, and ease of use during live interviews.

## Design Principles

1. **Minimal Distraction**: UI elements are subtle and non-intrusive
2. **Quick Glance**: Information is scannable in 2-3 seconds
3. **Accessibility**: High contrast, readable fonts, keyboard navigation
4. **Responsive**: Adapts to different screen sizes
5. **Customizable**: User can adjust position, size, transparency

**Characteristics**:
- Appear in corner (configurable)
- Auto-dismiss after 3-5 seconds
- Non-intrusive, subtle animation
- Stack vertically if multiple

## Responsive Design Considerations

### Mobile/Tablet Adaptation
- Overlay becomes full-screen modal on small screens
- Sidebar collapses to bottom sheet
- Touch-optimized button sizes
- Simplified controls

### Large Screen Optimization
- Overlay can be positioned anywhere
- Multiple suggestions can be visible
- Enhanced analytics visualization

## Accessibility Features

1. **Keyboard Navigation**:
   - Tab through all interactive elements
   - Enter/Space to activate
   - Escape to close/dismiss

2. **Screen Reader Support**:
   - ARIA labels on all elements
   - Live regions for dynamic content
   - Semantic HTML structure

3. **Visual Accessibility**:
   - High contrast mode option
   - Adjustable font sizes
   - Color-blind friendly palette

4. **Reduced Motion**:
   - Respects prefers-reduced-motion
   - Minimal animations option

## Color Scheme

### Dark Theme (Default)
- Background: #1a1a1a (semi-transparent)

- Accent: #4a9eff (blue)
- Success: #4caf50 (green)
- Warning: #ff9800 (orange)
- Error: #f44336 (red)

### Light Theme
- Background: #ffffff (semi-transparent)
- Text: #1a1a1a
- Accent: #1976d2 (blue)
- Similar accent colors with adjusted brightness
